Pulsar stats

分区主题

统计信息Description
msgRateIn所有本地发布者和复制发布者的发布频率总和，以消息/秒为单位。
msgThroughputIn与 msgRateIn 相同，但以字节/秒作为单位，而不是消息/秒。
msgRateOut所有本地消费者和复制消费者的 dispatch 频率总和，以消息/秒为单位。
msgThroughputOut与 msgRateOut 相同，但以字节/秒为单位，而不是消息/秒。
averageMsgSize此发布者在最后一段时间间隔内发布消息的平均大小，以字节为单位。
storageSize此主题的所有 ledger 存储大小总和。
publishersThe list of all local publishers into the topic. Publishers can be anywhere from zero to thousands.
producerId此主题上此生产者的内部标识符。
producerName此生产者的内部标识符，由客户端类库生成。
address此生产者连接的 IP 地址和源端口。
connectedSince此生产者创建或最后一次重新连接的时间戳。
subscriptions该主题的所有本地订阅列表。
my-subscription此订阅的名称 (由客户端定义)。
msgBacklog此订阅的消息积压数量。
type此订阅的类型。
msgRateExpired此订阅中消息由于 TTL 的原因没有被分发，而是被丢弃的频率。
consumers此订阅的已连接消费者列表。
consumerName此消费者的内部标识符，由客户端类库生成。
availablePermits此消费者在客户端监听队列中所能容纳的最大消息数。 值为0意味着客户端类库的队列已经满了，receive()不再被调用。 非零值意味着该消费者已经准备好接收消息。
replication提供此主题 cross-colo 复制的统计信息。
replicationBacklog出站复制积压的消息数。
connected出站复制器是否已连接。
replicationDelayInSeconds如果已连接，最老的消息经过多久的延迟才经由此连接发出。
inboundConnection通过发布者连接从远程集群与此 broker 进行连接的 broker 的 IP 和端口。
inboundConnectedSinceThe TCP connection being used to publish messages to the remote cluster. If no local publishers are connected, this connection is automatically closed after a minute.

主题

统计信息Description
entriesAddedCounter从此 broker 读取此主题后发布的消息。
numberOfEntries正被追踪的消息总数。
totalSize所有消息的存储大小总和，以字节为单位。
currentLedgerEntries当前开放写的 ledger中，写入的消息总数。
currentLedgerSize当前开放写的 ledger 中，写入消息的大小，以字节为单位。
lastLedgerCreatedTimestamp上一个 ledger 的创建时间。
lastLedgerCreationFailureTimestamp上一次创建 ledger 失败的时间。
waitingCursorsCount跟上消息发布的进度，正在等待新消息发布的游标数量。
pendingAddEntriesCount有写入请求（异步）并且请求正在等待完成的消息数。
lastConfirmedEntryThe ledgerid:entryid of the last message successfully written. If the entryid is -1, then the ledger is opened or is being currently opened but has no entries written yet.
stateThe state of the cursor ledger. Open means you have a cursor ledger for saving updates of the markDeletePosition.
ledgers本主题保存消息的所有 ledger 的有序列表。
cursorsThe list of all cursors on this topic. Every subscription you saw in the topic stats has one.
markDeletePositionThe ack position: 最后一条被订阅者确认收到的消息。
readPosition订阅者阅读消息的最新位置。
waitingReadOp当订阅者已经读取了发布到主题的最新消息，并且等待新消息发布时，值为 true。
pendingReadOps已经发往 BookKeeper 的读请求数量计数器。
messagesConsumedCounter从 broker 载入本主题后此游标已确认的消息总数。
cursorLedger用于持久化存储当前 markDeletePosition 的 ledger。
cursorLedgerLastEntry最后一个用于持久化存储当前 markDeletePosition 的 entryid。
individuallyDeletedMessages如果消息的确认是乱序的，则显示在 markDeletePosition 和 read-position 范围内的已确认消息。
lastLedgerSwitchTimestamp游标 ledger 的上一次回滚时间。
