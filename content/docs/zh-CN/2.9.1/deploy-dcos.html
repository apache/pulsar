Deploy Pulsar on DC/OS

Tips
To enable all built-in Pulsar IO connectors in your Pulsar deployment, we recommend you use apachepulsar/pulsar-all image instead of apachepulsar/pulsar image; the former has already bundled all built-in connectors.

DC/OS (the DataCenter Operating System) is a distributed operating system for deploying and managing applications and systems on Apache Mesos. DC/OS is an open-source tool created and maintained by Mesosphere.

Apache Pulsar 可作为Marathon Application Group，它以可管理的集合方式运行多个应用程序。

先决条件

在 DC/OS 上运行 Pulsar 之前，您需要准备好环境。

  • DC/OS 版本 1.9 或更高

  • 一个至少有三个代理节点的DC/OS cluster

  • 已安装 DC/OS CLI tool

  • 来自 Pulsar GitHub repo 的PulsarGroups.json 配置文件。

    $ curl -O https://raw.githubusercontent.com/apache/pulsar/master/deployment/dcos/PulsarGroups.json

DC/OS 管理的 Mesos 集群中的每个节点必须至少有：

  • 4 个 CPU
  • 4 GB 内存
  • 60 GB 总磁盘

或者，您可以相应地改变PulsarGroups.json中的配置，以匹配您的DC/OS集群中的资源。

使用 DC/OS 命令界面部署 Pulsar

通过此命令在 DC/OS 上部署 Pulsar：

$ dcos marathon group add PulsarGroups.json

此命令在三组中部署 Docker 容器实例，构成了一个 Pulsar 集群：

  • 3 个 bookie（每个代理节点上有 1 个 bookie 和 1 个 bookie recovery实例）
  • 3 个 Pulsar broker（每个节点 1 个 broker 和 1 个 admin 实例）
  • 1 个 Prometheus 实例 和 1 个 Grafana 实例

当你运行DC/OS时，一个ZooKeeper集群将在master.mesos:2181上运行，因此你不需要单独安装或启动ZooKeeper。

After executing the dcos command above, click the Services tab in the DC/OS GUI interface, which you can access at http://m1.dcos in this example. 正常情况下，可以看到几个正在部署的应用程序。

BookKeeper 组

To monitor the status of the BookKeeper cluster deployment, click the bookkeeper group in the parent pulsar group.

此时，3 个 bookies状态应该显示为绿色，表示 bookie 已经部署成功并且正在运行。

您还可以单击每个 bookie 实例以获取更详细的信息，例如 bookie 运行日志。

若要显示 ZooKeeper 中的 BookKeeper 信息，可以查看 http://m1.dcos/exhibitor。 在此例中，3 个 bookie 在 available 目录下。

Pulsar broker 组

Similar to the BookKeeper group above, click brokers to check the status of the Pulsar brokers.

您还可以单击每个Broker实例以获取更详细的信息，例如Broker运行日志。

ZooKeeper 中的 Broker 集群信息也可以通过 Web UI 获得。 在此例中，可以看到 loadbalance 和 managed-ledgers 目录已经创建。

监控组

The monitory group consists of Prometheus and Grafana.

Prometheus

点击进入 prom 实例，得到 Prometheus 的访问地址，此例中是 192.168.65.121:9090。

If you click that endpoint, you can see the Prometheus dashboard. All the bookies and brokers are listed on http://192.168.65.121:9090/targets.

Grafana

点击 grafana 来获取 Grafana 的地址，此例中是 192.168.65.121:3000。

如果单击该地址，则可以访问 Grafana 仪表板。

在 DC/OS 上运行简单的 Pulsar 消费者和生产者

现在已经完全部署好了 Pulsar 集群，接下来就可以运行一个简单的消费者和生产者来展示 DC/OS 上 Pulsar 的运作。

下载并准备 Pulsar Java 教程

可以使用 git clone 下载 Pulsar Java tutorial 仓库。 该 repo 包含了简单的 Pulsar 消费者和生产者（在此 repo 的 README 文件中找到更多的信息）。

$ git clone https://github.com/streamlio/pulsar-java-tutorial

将文件ConsumerTutorial.java和文件ProducerTutorial.java的 SERVICE_URL从pulsar://localhost:6650修改为pulsar://a1.dcos:6650。

pulsar://a1.dcos:6650 端点用于 broker 服务。 通过从 DC/OS GUI 中获取每个 broker 实例的端点详细信息。 a1.dcos 是作为运行Broker的 DC/OS 客户端代理，您可以将其替换为客户端代理IP地址。

现在，您可以在ProducerTutorial.java文件中的 main 方法中将消息编号从 10 更改为 10000000 以生成更多消息。

然后，您可以使用下面的命令编译项目代码：

$ mvn clean package

运行消费者和生产者

执行此命令来运行消费者：

$ mvn exec:java -Dexec.mainClass="tutorial.ConsumerTutorial"

运行此命令来运行生产者：

$ mvn exec:java -Dexec.mainClass="tutorial.ProducerTutorial"

您可以通过 DC/OS GUI 来查看生产者生产消息和消费者消费消息。

查看 Grafana metric 输出

当生产者和消费者运行时，可以从 Grafana 中访问运行 metrics 信息。

卸载 Pulsar

通过以下两种方式可以随时从 DC/OS 停止和卸载 pulsar 应用程序。

  1. Click the three dots at the right end of Pulsar group and choose Delete on the DC/OS GUI.

  2. 运行下面的命令：

    $ dcos marathon group remove /pulsar
