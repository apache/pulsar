Managing non-persistent topics

非持久topic可以用于仅想消费实时发布的消息，而不需要保证持久性的系统。使用非持久topic还可以减少消息发布的延迟，因为没有了持久化消息的开销。

下面所有说明和命令中，topic名称的结构为：

persistent://tenant/namespace/topic

非持久化topic资源

获取统计信息

给出非持久化topic的统计数据

  • ** msgRateIn **本地和复制的发布者每秒发布消息的速率

  • msgThroughputIn：同上，但不是每秒消息数，而是每秒发布消息的byte

  • ** msgRateOut **：本地和复制的消费者每秒分发消息的数量之和

  • msgThroughputOut：同上，但不是每秒的消息数量而是每秒消息的byte

  • ** averageMsgSize **：最后一次间隔内发布消息的平均byte大小

  • publishers：所有本地发布者列表，可以是零个，也可以是上千个

  • averageMsgSize：此发布者在最后一个间隔内发布消息的平均byte大小

  • producerId：此topic的producer的内部标识

  • producerName：客户端类库生成的此producer的内部标识符

  • address：此生产者的连接ip和源端口

  • ** connectedSince **：创建或上次重新连接该生产者的时间戳

  • ** subscriptions **: 该主题所有本地订阅的列表

  • my-subscription: 此订阅的名称 (客户端定义)

  • type：此订阅的类型

  • ** consumers **：此订阅连接的consumer列表

  • ** consumerName **：客户端类库生成的此consumer的内部标识符

  • availablePermits: 此consumer在客户端类库的监听队列中可以容纳的消息数量。 值为0意味着客户端类库的队列已经满了，receive()不再被调用。 非零值意味着该消费者已经准备好接收消息。

  • ** replication **: 该topic交叉复制的统计信息。

  • connected: 外部复制器是否已经连接

  • inboundConnection: 连接到此Broker的远程集群发布者broker的IP和端口。

  • inboundConnectedSince：正在被用来发布消息到远程集群的TCP连接。 如果没有连接上的本地发布者，连接将会在一分钟后关闭。

  • msgDropRate：发布者的broker只允许配置数量以内的消息进行传输，他会丢弃超过阈值的所有消息。 如果遇到不可用的限制或者连接不可写的情况，Broker也会丢弃订阅的消息。

pulsar-admin

可以使用stats命令来获取Topic的统计信息。

$ pulsar-admin non-persistent stats \
  non-persistent://test-tenant/ns1/tp1 \

REST API

GET /admin/v2/non-persistent/:tenant/:namespace/:topic/stats

Java

String topic = "non-persistent://my-tenant/my-namespace/my-topic";
admin.nonPersistentTopics().getStats(topic);

创建分区topic

Pulsar中，分区topic需要显示创建。当你创建分区topic，需要同时提供名称以及分区的数量。

pulsar-admin

$ bin/pulsar-admin non-persistent create-partitioned-topic \
  non-persistent://my-tenant/my-namespace/my-topic \
  --partitions 4

REST API

PUT /admin/v2/non-persistent/:tenant/:namespace/:topic/partitions

Java

String topicName = "non-persistent://my-tenant/my-namespace/my-topic";
int numPartitions = 4;
admin.nonPersistentTopics().createPartitionedTopic(topicName, numPartitions);

获取元数据

你可以以JSON对象来获取分区topic关联的元数据。目前可用的元数据字段如下：

字段含义
partitionsTopic划分的分区数量

pulsar-admin

$ pulsar-admin non-persistent get-partitioned-topic-metadata \
  non-persistent://my-tenant/my-namespace/my-topic
{
  "partitions": 4
}

REST API

GET /admin/v2/non-persistent/:tenant/:namespace/:topic/partitions

Java

String topicName = "non-persistent://my-tenant/my-namespace/my-topic";
admin.nonPersistentTopics().getPartitionedTopicMetadata(topicName);

  non-persistent://test-tenant/ns1/tp1 \

PUT /admin/v2/non-persistent/:tenant/:namespace/:topic/unload

admin.nonPersistentTopics().unload(topic);
