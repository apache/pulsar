Elasticsearch sink connector

Elasticsearch sink 连接器从 Pulsar 主题拉取消息并将消息持久化到索引中。

功能

处理数据

从 Pulsar 2.9.0 开始，Elasticsearch sink 连接器有以下工作方式。 你可以选择其中一个。

配置项说明
Raw processingSink 从主题中读取并将原始内容传递给 Elasticsearch。

This is the default behavior.

Raw processing was already available in Pulsar 2.8.x.
Schema awareThe sink uses the schema and handles AVRO, JSON, and KeyValue schema types while mapping the content to the Elasticsearch document.

If you set schemaEnable to true, the sink interprets the contents of the message and you can define a primary key that in turn used as the special _id field on Elasticsearch.



This allows you to perform

UPDATE, INSERT, and DELETE operations to Elasticsearch driven by the logical primary key of the message.

This is very useful in a typical Change Data Capture scenario in which you follow the changes on your database, write them to Pulsar (using the Debezium adapter for instance), and then you write to Elasticsearch.

You configure the mapping of the primary key using the primaryFields configuration entry.

The DELETE operation can be performed when the primary key is not empty and the remaining value is empty. 使用 nullValueAction 来配置此行为。 默认配置只是忽略这样的空值。

映射多个索引

从 Pulsar 2.9.0 起， indexName 属性不再需要。 如果省略它，sink 将在写入 Pulsar 主题名称后写入索引名称。

启用批量写入

从 Pulsar 2.9.0 开始，你可以将 bulkEnabled 属性设置为 true 来使用批量写入。

通过 TLS 启用安全连接

从 Pulsar 2.9.0 开始，你可以使用 TLS 启用安全连接。

示例

在使用 Elasticsearch sink 连接器之前，你需要通过以下方法之一创建配置文件。

Configuration

Elasticsearch 之后 6.2 版本

  • JSON

    {
    "elasticSearchUrl": "http://localhost:9200",
    "indexName": "my_index",
    "username": "scooby",
    "password": "doobie"
}

  • YAML

    configs:
    elasticSearchUrl: "http://localhost:9200"
    indexName: "my_index"
    username: "scooby"
    password: "doobie"

Elasticsearch 6.2 之前的版本

  • JSON

    {
    "elasticSearchUrl": "http://localhost:9200",
    "indexName": "my_index",
    "typeName": "doc",
    "username": "scooby",
    "password": "doobie"
}

  • YAML

    configs:
    elasticSearchUrl: "http://localhost:9200"
    indexName: "my_index"
    typeName: "doc"
    username: "scooby"
    password: "doobie"

用法

  1. 启动一个单节点的 Elasticsearch 集群

    $ docker run -p 9200:9200 -p 9300:9300 \
    -e "discovery.type=single-node" \
    docker.elastic.co/elasticsearch/elasticsearch:7.13.3

  2. 以独立模式启动本地的 Pulsar 服务。

    $ bin/pulsar standalone

    确保 NAR 文件在 connectors/pulsar-io-elastic-search-2.10.0.nar 中可用。

  3. 使用以下方法之一在本地运行模式中启动 Pulsar Elasticsearch 连接器。

    • 使用前面显示的 JSON 配置。

      $ bin/pulsar-admin sinks localrun \
    --archive connectors/pulsar-io-elastic-search-2.10.0.nar \
    --tenant public \
    --namespace default \
    --name elasticsearch-test-sink \
    --sink-config '{"elasticSearchUrl":"http://localhost:9200","indexName": "my_index","username": "scooby","password": "doobie"}' \
    --inputs elasticsearch_test

    • 使用前面显示的 YAML 配置文件。

      $ bin/pulsar-admin sinks localrun \
    --archive connectors/pulsar-io-elastic-search-2.10.0.nar \
    --tenant public \
    --namespace default \
    --name elasticsearch-test-sink \
    --sink-config-file elasticsearch-sink.yml \
    --inputs elasticsearch_test

  4. 向 topic 生产消息。

    $ bin/pulsar-client produce elasticsearch_test --messages "{\"a\":1}"

  5. 在 Elasticsearch 上查看创建的文档。

    • 刷新索引 
          $ curl -s http://localhost:9200/my_index/_refresh
    • 查询文档 
          $ curl -s http://localhost:9200/my_index/_search
      你可以看到先前在 Pulsar 上生产的消息已经成功写入到 Elasticsearch 中了。 
      {"took":2,"timed_out":false,"_shards":{"total":1,"successful":1,"skipped":0,"failed":0},"hits":{"total":{"value":1,"relation":"eq"},"max_score":1.0,"hits":[{"_index":"my_index","_type":"_doc","_id":"FSxemm8BLjG_iC0EeTYJ","_score":1.0,"_source":{"a":1}}]}}
