在本地搭建Pulsar集群

为了在本地开发和测试，你可以在本机以独立模式运行Pulsar。 独立模式包含Pulsar broker、必须的Zookeeper和BookKeeper组件，都运行在一个Java虚拟机(JVM) 进程之中。

Pulsar运行在生产环境?

如果你想在生产环境运行完整的Pulsar，请参考 Pulsar的集群部署

手工运行Pulsar

System requirements

Pulsar当前在MacOS 和 Linux下可用. 如果要使用Pulsar，你需要先安装Java 8.

安装 Pulsar

开始运行Pulsar之前，请先用下列几种方式下载二进制包:

  • 通过单击下面的链接并从 Apache 镜像下载该版本:

    • Pulsar 2.2.0 binary release

  • 从Pulsar 下载页

  • 从 Pulsar github releases 页面

  • 使用wget：

    $ wget https://archive.apache.org/dist/pulsar/pulsar-2.2.0/apache-pulsar-2.2.0-bin.tar.gz

下载包后, 解压它和 cd 到生成的目录中:

$ tar xvfz apache-pulsar-2.2.0-bin.tar.gz
$ cd apache-pulsar-2.2.0

软件包包含的内容:

Pulsar二进制包包含下列目录:

目录包含
binPilsar的命令行工具,比如pulsar 和 pulsar-admin
confPulsar的配置文件,包含broker的配置,ZooKeeper 的配置 等等
examples一个Java JAR包，包含 Pulsar Functions的例子
libPulsar用到的JAR 包
licenses开源许可文件. .txt格式, 适用于Pulsar 代码库的各种组件

这些目录将在你运行Pulsar的时候生成:

目录包含
dataZookeeper和BookKeeper使用的数据存储目录
instances为Pulsar Functions创建的组件
logs安装时创建的日志

Installing Builtin Connectors

Since release 2.1.0-incubating, Pulsar releases a separate binary distribution, containing all the builtin connectors. If you would like to enable those builtin connectors, you can download the connectors tarball release in one of the following ways:

  • 通过单击下面的链接并从 Apache 镜像下载该版本:

    • Pulsar IO Connectors 2.2.0 release

  • 从Pulsar 下载页

  • 从 Pulsar github releases 页面

  • 使用wget：

    $ wget https://archive.apache.org/dist/pulsar/pulsar-2.2.0/apache-pulsar-io-connectors-2.2.0-bin.tar.gz

Once the tarball is downloaded, in the pulsar directory, untar the io-connectors package and copy the connectors as connectors in the pulsar directory:

$ tar xvfz /path/to/apache-pulsar-io-connectors-2.2.0-bin.tar.gz

// you will find a directory named `apache-pulsar-io-connectors-2.2.0` in the pulsar directory
// then copy the connectors

$ cd apache-pulsar-io-connectors-2.2.0/connectors connectors

$ ls connectors
pulsar-io-aerospike-2.2.0.nar
pulsar-io-cassandra-2.2.0.nar
pulsar-io-kafka-2.2.0.nar
pulsar-io-kinesis-2.2.0.nar
pulsar-io-rabbitmq-2.2.0.nar
pulsar-io-twitter-2.2.0.nar
...

NOTES

如果你是在裸金属(bare metal) 的集群中运行Pulsar，那么你需要确保connectors解压到在所有的broker的pulsar 目录中(或者如果你将Pulsar functions运行在单独的worker集群中，你也可以将其解压到每一个运行function的pulsar目录中)

如果你是在Docker中运行Pulsar 或者 通过Docker镜像部署的Puslar(比如:K8S 或者 DCOS), 你可以使用apachepulsar/pulsar-all 镜像，而不是apachepulsar/pulsar. apachepulsar/pulsar-all image has already bundled all builtin connectors.

启动集群

一旦你在本地下载了最新的发行版，就可以使用 bin目录下的 pulsar 命令启动一个本地集群了，并且指定你需要运行在独立模式下:

$ bin/pulsar standalone

如果Pulsar已经成功启动，你应该会看到INFO级别的日志消息，如下所示：

2017-06-01 14:46:29,192 - INFO  - [main:WebSocketService@95] - Global Zookeeper cache started
2017-06-01 14:46:29,192 - INFO  - [main:AuthenticationService@61] - Authentication is disabled
2017-06-01 14:46:29,192 - INFO  - [main:WebSocketService@108] - Pulsar WebSocket Service started

自动创建的命名空间

启动本地独立模式的集群时,Pulsar会自动的创建public/default 命名空间,您可以用于开发的目的. 所有Pulsar的topic主题都在命名空间中进行管理。 更多信息, 请参考 主题。

测试集群

Pulsar提供了一个名为pulsar-client 的CLI工具,它使你可以执行类似给正在运行的Pulsar集群的某个topic发消息等操作. 这条命令将发送一条简单的消息hello-pulsar给my-topic topic:

$ bin/pulsar-client produce my-topic --messages "hello-pulsar"

如果消息成功发送给了topic，你应该会在pulsar-client日志中找到一个确认， 类似：

13:09:39.356 [main] INFO  org.apache.pulsar.client.cli.PulsarClientTool - 1 messages successfully produced

不需要显式的创建topic

你可能注意到了，我们并没有显式地创建刚刚发送hello-pulsar 这条消息的topic my-topic。 如果你往一个不存在的topic发送消息，Pulsar会自动为你创建topic。

在本地使用Pulsar客户端

Pulsar当前为Java, Python, and C++ 提供了客户端. 如果你运行的是本地独立集群，你可以使用下面这些URL中的一个，从而和你的Pulsar集群交互:

  • http://localhost:8080
  • pulsar://localhost:6650

以下是使用Java客户端创建producer的例子:

String localClusterUrl = "pulsar://localhost:6650";

PulsarClient client = PulsarClient.builder().serviceURL(localClusterUrl).build();
Producer<byte[]> producer = client.newProducer().topic("my-topic").create();

以下是Python producer的例子:

import pulsar

client = pulsar.Client('pulsar://localhost:6650')
producer = client.create_producer('my-topic')

最后是 C++ producer的例子:

Client client("pulsar://localhost:6650");
Producer producer;
Result result = client.createProducer("my-topic", producer);
if (result != ResultOk) {
    LOG_ERROR("Error creating producer: " << result);
    return -1;
}
