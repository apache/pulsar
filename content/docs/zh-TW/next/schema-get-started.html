開始

本章介紹了 Pulsar schema，並解釋了為什麼它們很重要。

Schema 註冊管理

對於圍繞信息系統如 Pulsar 搭建的應用來說，資料/事件類型安全 (type safety) 非常的重要。

Producer 和 consumer 需要某種機制在 topic 級別上協調資料/事件的類型，以避免出現各種潛在問題。 例如，序列化和反序列化問題。

應用程序通常採用以下方法之一來保證信息傳遞中的類型安全。 這兩種方法都被 Pulsar 支持，你可以在 topic 層級的基礎上，自由選擇採用哪一種，或者混用。

Note

Currently, the Pulsar schema registry is only available for the Java client, CGo client, Python client, and C++ client.

客户端方法

Producer 和 consumer 要負責了解如何對 topic 裡的信息進行序列化與反序列，同時得知道這個 topic 裡的信息是那一種資料/事件類型。

如果 producer 正在發送有關溫度傳感器數據到topic-1，如果訂閱該 topic 的 consumer 在嘗試將其解析為濕度傳感器讀數時會遇到麻煩。

Producer 和 consumer 可以發送和接收包含原始字節數組的信息，並透過其它外部的手法來進行資料/事件類型的安全性強制措施。

服務器端方法

Producer 和 consumer 通知信息系統該 topic 傳輸哪種資料/事件數據類型。

With this approach, the messaging system enforces type safety and ensures that producers and consumers remain synced.

Pulsar 具有內建的 schema registry 服務，使客戶端可以遵循在 topic 層定義的資料/事件類型來發佈與消費。 Those schemas dictate which data types are recognized as valid for that topic.

為什麼要使用 schema

啟用 schema 的功能後，Pulsar 會解析數據，它將 bytes 作為輸入並發送 bytes 作為輸出。 在對信息數據進行解析時可能會遇到解析異常，這種異常主要發生在以下幾種情況：

  • 該字段不存在

  • 字段類型已更改（例如，將string更改為int）

有幾種方法可以預防和解決這些異常，例如，可以在解析錯誤時捕捉異常，但這會使得代碼難以維護；或者可以採用 schema 管理系統來執行 schema 演化，而且不破壞下游應用程序，並在不同程式語言的結構上以最大程度地強化類型安全。Pulsar 解決這個問題的方案就是 Pulsar schema。

使用 Pulsar schema，你可以使用特定於語言的數據類型來構建和處理從 string 之類的簡單類型到更複雜的應用程序特定類型的信息。

Example

你可以使用 User 類別來定義發送到 Pulsar topic 的信息。

public class User {
    String name;
    int age;
}

當構建 producer 來使用 User 類別，可以指定 schema 或不指定。

不指定 Schema 情況

在不指定 schema 的情況下，則 producer 只能產生 byte[]類型的信息。 如果你有 POJO 類別，則需要在發送信息之前將 POJO 序列化為 byte[]。

Example

Producer<byte[]> producer = client.newProducer()
        .topic(topic)
        .create();
User user = new User("Tom", 28);
byte[] message = … // serialize the `user` by yourself;
producer.send(message);

指定 Schema 情況

如果你來構造生產者時指定 schema，則可以直接將類別實例發送進 topic，而不必擔心如何將 POJO 序列化為 byte[]。

Example

本範例在 producer 中指定使用 JSONSchema，你可以直接將 User 類別實例發送給 topic，而不必擔心如何將其序列化為 byte[]。

Producer<User> producer = client.newProducer(JSONSchema.of(User.class))
        .topic(topic)
        .create();
User user = new User("Tom", 28);
producer.send(user);

總結

當構建 producer 時指定 schema，就不需要在客戶端去思考如何將信息序列化為byte[]，Pulsar Schema 會在後台執行此工作。
