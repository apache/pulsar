部署和管理 Functions-worker

在使用 Pulsar Functions 之前，需要學習如何配置 Pulsar Functions worker 以及設定 Functions runtime。

Pulsar functions-worker 是在群集模式下運行 Pulsar Functions 的邏輯組件。 可以根據需要來選擇兩個選項之一。

  • 與 broker 運行在一起
  • 獨立的運行

注意
下圖中的--- Service Urls ---表示 Pulsar 服務URL，Pulsar客戶端和管理員使用這些 URL 連接到 Pulsar 群集。

Functions-worker 與 broker 運行在一起

下圖說明了將 functions-worker 與 broker 佈署在一起來運行。

要使 functions-worker 作為 broker 的一部分來運行，需要在 broker.conf 設定檔中將 functionsWorkerEnabled 設為 true。

functionsWorkerEnabled=true

當將 functionsWorkerEnabled 設為 true 時，這意味著將啟動 function-worker 作為 broker 的一部分。 你需要配置 conf/functions_worker.yml 設定檔來定義 functions_worker。

在通過 broker 運行 Functions-worker 之前，必須配置 Functions-worker，然後使用 broker 啟動它。

設定 Functions-Worker 與 broker 一起運行

在此模式下，由於 functions-worker 作為 broker 程序的一部分運行，因此大多數設定已經從 broker 程序配置繼承（例如，configurationStore 設定、身份驗證設定等）。

在此模式下配置 functions-worker 時，請注意以下必需的設定。

  • numFunctionPackageReplicas: 存儲 function packages 的副本數。 預設值為 1，這對於單機模式部署非常有用。 對於生產環境部署，要確保高可用性，請將其設定調整為大於 2。
  • pulsarFunctionsCluster: 將值設置為你的 Pulsar 群集名稱（與 broker 配置中的 clusterName 設定相同）。

如果在 BookKeeper 群集上啟用了身份驗證，請配置以下 BookKeeper 身份驗證設定。

  • bookkeeperClientAuthenticationPlugin: BookKeeper 客戶端身份驗證插件名稱。
  • bookkeeperClientAuthenticationParametersName: BookKeeper 客戶端身份驗證插件參數名稱。
  • bookkeeperClientAuthenticationParameters: BookKeeper 客戶端身份驗證插件參數值。

啟動內嵌 Functions-worker 的 broker

配置 functions_worker.yml 設定檔後，就可以啟動或重新啟動 broker。

然後，可以使用以下命令來驗證 functions-worker 是否運行良好。

curl <broker-ip>:8080/admin/v2/worker/cluster

輸入上述命令後，將返回集群中 active 的 function-workers 列表。 輸出類似於以下內容。

[{"workerId":"<worker-id>","workerHostname":"<worker-hostname>","port":8080}]

獨立運行 Function-worker

本節說明如何將 functions-worker 作為單獨的進程服務來運行。

注意
在這種模式下，請確保將 functionsWorkerEnabled 設定為 false，這樣就不會錯誤地使用 broker 來啟動 functions-worker。

設定 Function-worker 獨立運行

要單獨運行 function-worker，必須設定以下參數。

Worker 參數

  • workerId: 類型是字串。 它在集群中是唯一的，用於標識一台 worker 節點。
  • workerHostname: worker 節點的主機名。
  • workerPort: worker 節點網路監聽的端口。 如果不想定義，請將其保留為預設值。
  • workerPortTls: worker 節點網路監聽的 TLS 端口。 如果不想定義，請將其保留為預設值。

Function package 參數

  • numFunctionPackageReplicas: 存儲 function packages 的副本數。 預設值為 1。

Function metadata 參數

  • pulsarServiceUrl: Pulsar 群集的服務 URL。
  • pulsarWebServiceUrl: Pulser 群集的 Web 服務 URL。
  • pulsarFunctionsCluster: 將值設定為你的 Pulsar群集名稱（與 broker 配置中的 clusterName 設定相同）。

如果 broker 群集啟用了身份驗證，則應該配置身份驗證插件和參數，以使 function worker 可以與 broker 進行通信。

  • clientAuthenticationPlugin
  • clientAuthenticationParameters

Security 設定

如果要在 function worker 上啟用安全性功能，則*應該</ em>：

  • 啟動 TLS transport encryption
  • 啟用 Authentication Provider
  • 啟用 Authorization Provider
啟用 TLS transport encryption

要啟用 TLS傳輸加密，請配置以下設定。

tlsEnabled: true
tlsCertificateFilePath: /path/to/functions-worker.cert.pem
tlsKeyFilePath:         /path/to/functions-worker.key-pk8.pem
tlsTrustCertsFilePath:  /path/to/ca.cert.pem

有關 TLS加 密的詳細訊息，請參閱使用TLS的傳輸加密。

啟用 Authentication Provider

要在 Functions Worker 上啟用身份驗證，請配置以下設定。

注意
將*provider 列表</ em>替換為要啟用的provider。

authenticationEnabled: true
authenticationProviders: [ provider1, provider2 ]

對於 SASL Authentication provider，在 properties 下添加 saslJaasClientAllowedIds 和 saslJaasBrokerSectionName。

properties:
  saslJaasClientAllowedIds: .*pulsar.*
  saslJaasBrokerSectionName: Broker

對於 Token Authentication prodivder，在properties下添加所需要的設定。 有關更多詳細信息，請參見 Token Authentication。

properties:
  tokenSecretKey:       file://my/secret.key 
  # If using public/private
  # tokenPublicKey:     file:///path/to/public.key
啟用 Authorization Provider

對 Functions Worker 啟用授權功能，需要設定 authorizationEnabled 與 configurationStoreServers。 經由 authentication provider 連接到 configurationStoreServers 來取得命名空間授權策略。

authorizationEnabled: true configurationStoreServers: <configuration-store-servers>


你應該配置超級用戶角色列表。 超級用戶角色可以訪問任何管理 API。 以下是配置範例。

```yaml
superUserRoles:
  - role1
  - role2
  - role3

BookKeeper 認證方式

如果在 BookKeeper 群集上啟用了身份驗證，請配置以下 BookKeeper 身份驗證設定。

  • bookkeeperClientAuthenticationPlugin: BookKeeper 客戶端身份驗證插件名稱。
  • bookkeeperClientAuthenticationParametersName: BookKeeper 客戶端身份驗證的插件參數名稱。
  • bookkeeperClientAuthenticationParameters: BookKeeper 客戶端身份驗證的插件參數。

啟動 Functions-worker

完成配置functions_worker.yml設定文件後，可以使用以下命令啟動 functions-worker：

bin/pulsar functions-worker

為 Functions-worker 配置 Proxy

當在分開的群集中運行 functions-worker 與 broker 時，admin rest endpoints 將分別落在兩個群集。 functions、function-worker、source 與 sink endpoints 都將啟動在 functions-worker 集群, 然而其它的 endpoints 啟動在 broker 集群。 因此，需要在使用 pulsar-admin 時指定正確的服務 URL。

為了解決此不便之處，可以啟動 proxy 群集來相應路由 admin rest請求。 因此，將擁有一個用於 admin rest 服務的集中入口點。

如果你已經有 proxy 群集，請繼續閱讀。 如果之前沒有設置 proxy 群集，則可以按照說明來啟動 proxy 服務。

要在 proxy 中啟用 functions-worker 管理 request 的路由，可以在 proxy.conf 文件修改以下設定：

functionWorkerWebServiceURL=<pulsar-functions-worker-web-service-url>
functionWorkerWebServiceURLTLS=<pulsar-functions-worker-web-service-url>

Run-with-Broker 與 Run-separately 模式的比較

如上所述，你可以與 broker 一起運行 Function-worker，或單獨運行它。 而且，與 broker 一起運行 function-worker 則更方便。 但是，在單獨的群集中運行 function-worker 可以為在 Process 或 Thread 模式下運行的 function 提供更好的資源隔離。

針對你的案例來使用哪種模式，請參考以下準則進行確定。

在以下情況，請使用 Run-with-Broker 模式：

  • a) 如果在 Process 或 Thread 模式下運行 function 時，不需要考量資源隔離；
  • b) 如果 functions-worker 被設定跑在 Kubernetes 上來運行（Kubernetes 解決了資源隔離問題）。

在以下情況，請使用 Run-separately 模式：

  • a) 你沒有 Kubernetes 集群;
  • b) 如果想要分別運行 function 和 broker。

故障排除

錯誤信息: Namespace missing local cluster name in clusters list

Failed to get partitioned topic metadata: org.apache.pulsar.client.api.PulsarClientException$BrokerMetadataException: Namespace missing local cluster name in clusters list: local_cluster=xyz ns=public/functions clusters=[standalone]

出現以下兩種情況時，都會出現以上的錯誤訊息：

  • a) 使用 functionsWorkerEnabled = true 來啟動 broker，但是在 conf/functions_worker.yaml 文件中未將 pulsarFunctionsCluster 設定為正確的集群；
  • b) 使用 functionsWorkerEnabled = true 設定 geo-replicated Pulsar 集群，其中一個集群的 broker 運行良好，而另一個集群中的 broker 則運行不正常。

因應措施

如果發生以上任何一種情況，請按照以下說明來解決：

  1. 取得當前擁有 public/functions 命名空間的群集列表。
bin/pulsar-admin namespaces get-clusters public/functions
  1. 檢查群集是否在群集列表中。 如果集群不在列表中，請將其添加到列表中並更新集群列表。
bin/pulsar-admin namespaces set-clusters --cluster=<existing-clusters>,<new-cluster> public/functions
  1. 在 conf/functions_worker.yml 文件的pulsarFunctionsCluster 中設定正確的集群名稱。
