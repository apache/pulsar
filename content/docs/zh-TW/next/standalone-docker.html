使用 Docker 運行 Pulsar 單機集群

對於本地開發和測試，你可以在自己機器的 Docker 容器中以單機模式運行 Pulsar。

如果你還沒有安裝Docker，你可以下載社群版本並按照官網上不同作業系統的說明來進行安裝與操作。

在 Docker 中啟動 Pulsar

  • 如果你是 MacOS、Linux、Windows 用戶：

    $ docker run -it \
-p 6650:6650 \
-p 8080:8080 \
--mount source=pulsardata,target=/pulsar/data \
--mount source=pulsarconf,target=/pulsar/conf \
apachepulsar/pulsar:2.6.1 \
bin/pulsar standalone

關於此命令的一些注意事項： *數據，元數據和配置保留在 Docker volume中，以免容器重新啟動時都要再重新花時間做配置與初始化。 關於volume的詳細，你可以使用 docker volume inspect <sourcename> * 對於 Windows 上的 Docker，請務必配置它使用 Linux 容器

如果成功啟動了 Pulsar，將看到如下所示的` INFO </ code>級日誌消息：

`

提示

當你啟動本地單機集群時，Pular 將自動創建 public/default 命名空間。 這個預設的命名空間主要用於開發。 所有 Pulsar 的 topics 都在命名空間內進行管理。 更多相關內容请参考 Topics。

在 Docker 中運行 Pulsar

Pulsar 支持多種語言的客戶端： Java，Go，<a href =“ client-libraries-python.md“>Python 和C ++ 。 如果你正在運行本地單機群集，則可以使用以下的 URL 來連接到你的集群：

  • pulsar://localhost:6650
  • http://localhost:8080

下面的範例將帶領你使用Python快速地學會使用 Pulsar 客戶端 API。

直接從PyPI安裝 Pulsar Python 客戶端：

$ pip install pulsar-client

消費一條信息

創建 consumer 並訂閱 topic：

import pulsar

client = pulsar.Client('pulsar://localhost:6650')
consumer = client.subscribe('my-topic',
                            subscription_name='my-sub')

while True:
    msg = consumer.receive()
    print("Received message: '%s'" % msg.data())
    consumer.acknowledge(msg)

client.close()

產生一條信息

現在啟動 producer 用來發送一些測試信息：

import pulsar

client = pulsar.Client('pulsar://localhost:6650')
producer = client.create_producer('my-topic')

for i in range(10):
    producer.send(('hello-pulsar-%d' % i).encode('utf-8'))

client.close()

獲取 topic 統計資訊

在 Pulsar 中，可以使用 REST、Java 或命令行工具來控制系統的各個層面。 有關 API 的詳細說明，請參閱管理 API 概述。

以下範例使用 curl 來探查特定 topic 的統計資訊：

$ curl http://localhost:8080/admin/v2/persistent/public/default/my-topic/stats | python -m json.tool
