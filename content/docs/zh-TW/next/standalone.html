安裝 Pulsar 單機模式

對於本地開發和測試，你可以在自己的電腦上以單機模式來運行 Pulsar。 單機模式包含一個 Pulsar broker、必須的 Zookeeper 和 BookKeeper 組件，它們運行在一個單獨的 Java 虛擬機（JVM）進程之中。

Pulsar 運行在生產環境？

如果你想在生產環境運行完整的 Pulsar，請參考 Pulsar 的集群部署。

安裝 Pulsar 單機模式

本教程將講解安裝過程的每個步驟。

系統要求

Pulsar 目前可用於** MacOS 和 Linux **。 要使用 Pulsar，需要從 Oracle 下載中心安裝 Java 8。

提示

預設情況下，Pulsar 分配 2G JVM heap 內存來啟動。可以透過修改在conf/pulsar_env.sh文件中的PULSAR_MEM 參數來調整內存分配大小。這是傳遞給 JVM 的額外選項。

使用二進制發行版安裝 Pulsar

要開始使用 Pulsar，請透過以下方式下載二進制 tarball 版本：

  • 從 Apache 鏡像站下載(Pulsar 2.6.0 二進制版本)

  • 從 Pulsar 官網下載頁

  • 從 Pulsar 的 Github 發布頁面下載

  • 使用wget：

    $ wget https://archive.apache.org/dist/pulsar/pulsar-2.6.0/apache-pulsar-2.6.0-bin.tar.gz

下載 tarball 壓縮包後，解壓縮並使用cd命令切換 到結果目錄：

$ tar xvfz apache-pulsar-2.6.0-bin.tar.gz
$ cd apache-pulsar-2.6.0

軟體包包含的內容:

Pulsar 二進制包包含下列目錄:

目錄包含
binPulsar 的命令行工具，比如 pulsar 和 pulsar-admin。
confPulsar 的配置文件，包含 broker 配置,ZooKeeper 配置 等等。
examples一個 Java JAR 包，包含 Pulsar Functions 的範例。
libPulsar 使用到的 JAR 文件
licenses開源許可文件 .txt格式, 適用於 Pulsar 代碼庫 裡的各種組件。

以下目錄是在運行 Pulsar 的時候生成。

目錄包含
dataZooKeeper 和 BookKeeper 使用的數據存儲目錄
instances為 Pulsar Functions 創建的組件。
logs安裝時生成的日誌文件目錄

提示

如果要使用內建連接器和分層存儲卸載插件，可以按照以下說明進行安裝：

  • 安裝內建連接器（可選）
  • 安裝分層存儲卸載插件（可選）

否則，請跳過此步驟並執行下一步啟動 Pulsar 單機模式。 無需安裝內建連接器和分層存儲卸載插件也可以成功安裝與運行 Pulsar。

安裝內建 Source/Sink 連接器（可選）

自從2.1.0-incubating 版本，Pulsar 會為不同的 Source/Sink 連接器發布了一個單獨的二進制版。你可依實際的需要去下載這些內建的連接器。 要啟用這些 內建 連接器，首先到下列的下載點去下載所需要的連接器：

  • 從Apache鏡像站下載 Pulsar IO 連接器 {{pulsar：version}} 版本

  • 從 Pulsar 官網下載頁

  • 從 Pulsar 的 Github 發布頁面下載

  • 使用wget：

    $ wget https://archive.apache.org/dist/pulsar/pulsar-2.6.0/connectors/{connector}-2.6.0.nar

下載完 nar 文件後，將文件複製到 pulsar 目錄中的 connectors 目錄。 例如，如果你下載 pulsar-io-aerospike-2.6.0.nar 連接器檔案，請輸入以下命令：

$ mkdir connectors
$ mv pulsar-io-aerospike-2.6.0.nar connectors

$ ls connectors
pulsar-io-aerospike-2.6.0.nar
...

注意

  • 如果在裸機集群中運行 Pulsar，請確保將` connectors </ code> tarball 解壓縮到 broker 上面的 pulsar 目錄（如果正在為 Pulsar Functions 運行單獨的工作程序集群，則解壓縮到 function-worker 的 pulsar 目錄中）。

  • 如果在 Docker 中運行 Pulsar 或使用 Docker 鏡像部署 Pulsar（例如 K8S 或 DCOS ），則可以使用 apachepulsar/pulsar-all `鏡像代替` apachepulsar/pulsar `鏡像。 `apachepulsar/pulsar-all` 鏡像己經包含了[所有內建連接器](/docs/zh-TW/next/io-overview#working-with-connectors)。

    安裝分層存儲卸載插件（可選）

    提示

    自從2.1.0-incubating 版本，Pulsar 會為不同的 Source/Sink 連接器發布了一個單獨的二進制版。你可依實際的需要去下載這些內建的連接器。 要啟用分層存儲功能，請按照以下說明進行操作；否則請跳過本節。

    要開始使用 分層存儲卸載插件，你需要透過以下任一種方式在每個 broker 節點安裝這個插件：

    • 從 Apache鏡像站下載 Pulsar IO 連接器{{pulsar：version}}版本

    • 從 Pulsar 官網下載頁

    • 從 Pulsar 的 Github 發布頁面下載

    • 使用wget：

      $ wget https://archive.apache.org/dist/pulsar/pulsar-2.6.0/apache-pulsar-offloaders-2.6.0-bin.tar.gz

    下載完壓縮包後，解壓縮並將"分層存儲卸載插件"複製到 pulsar 目錄中的offloaders目錄：

    $ tar xvfz apache-pulsar-offloaders-2.6.0-bin.tar.gz

// you will find a directory named `apache-pulsar-offloaders-2.6.0` in the pulsar directory
// then copy the offloaders

$ mv apache-pulsar-offloaders-2.6.0/offloaders offloaders

$ ls offloaders
tiered-storage-jcloud-2.6.0.nar

    有關如何配置分層存儲的更多內容，請參見 分層存儲指南。

    注意

    • 如果在裸機集群中運行 Pulsar，請確保在每個 broker 的 pulsar 目錄中將offloaders tarball 解壓縮。

    • 如果在 Docker 中運行 Pulsar 或使用 Docker 鏡像部署 Pulsar（例如 K8S 或 DCOS ），則可以使用apachepulsar/pulsar-all鏡像代替apachepulsar/pulsar鏡像。 apachepulsar/pulsar-all 鏡像己經包含了所有內建連接器。

    啟動 Pulsar 單機模式

    取得 Pulsar 發行版的最新本地副本後，可以使用 pulsar 命令，該命令位於bin目錄中，並且使用參數 standlane 指定要以單機模式啟動 Pulsar。

    $ bin/pulsar standalone

    如果成功啟動了 Pulsar，將看到如下所示的INFO級日誌消息：

    2017-06-01 14:46:29,192 - INFO  - [main:WebSocketService@95] - Configuration Store cache started
2017-06-01 14:46:29,192 - INFO  - [main:AuthenticationService@61] - Authentication is disabled
2017-06-01 14:46:29,192 - INFO  - [main:WebSocketService@108] - Pulsar WebSocket Service started

    提示

    • 該服務在你可以直接控制的終端上運行。 如果需要使用其他命令，請打開一個新的終端窗口。你還可以使用pulsar-daemon start standalone命令將服務作為後台進程運行。 有關更多信息，請參見 pulsar-daemon 。

    • 預設配置沒有啟用加密、身份驗證或授權。 因此無需授權即可從遠端來訪問 Apache Pulsar。 請詳閱安全概述文檔來保護所部署的 Pulsar。

    • 啟動本地單機群集時，Pulsar 會自動創建一個 public/default 的命名空間。 這個預設的命名空間主要用於開發。 所有 Pulsar 的 topics 都在命名空間內進行管理。 更多相關內容请参考 Topics。

    Pulsar 單機模式的操作

    Pulsar 提供一個名為 pulsar-client 的 CLI 工具。 Pulsar-client 工具能夠在運行的 Pulsar 集群中對某一個指定topic來進行信息的發佈與消費。

    消費一條信息

    以下命令會從 my-topic 主題裡消費一個信息，訂閱名稱為 first-subscription ：

    $ bin/pulsar-client consume my-topic -s "first-subscription"

    如果topic訂閱成功，應該會在 pulsar-client 中看到類似以下的日誌：

    09:56:55.566 [pulsar-client-io-1-1] INFO  org.apache.pulsar.client.impl.MultiTopicsConsumerImpl - [TopicsConsumerFakeTopicNamee2df9] [first-subscription] Success subscribe new topic my-topic in topics consumer, partitions: 4, allTopicPartitionsNumber: 4

    提示

    如你所見，對於我們要消費的 topic 並沒有事先地去創建。 當要從不存在的 topic 消費一條信息時，Pulsar 會自動創建該 topic。 對發佈信息到不存在的 topic 時，Pulsar 也會自動創建該 topic。

    產生一條信息

    以下命令將發送一條簡單的信息 hello-pulsar 給 my-topic 主題:

    $ bin/pulsar-client produce my-topic --messages "hello-pulsar"

    如果信息成功發送給了 topic，應該會在 pulsar-client 中看到類似以下的日誌：

    13:09:39.356 [main] INFO  org.apache.pulsar.client.cli.PulsarClientTool - 1 messages successfully produced

    停止 Pulsar 單機模式

    按 Ctrl+C 停止單機模式運行的 Pulsar。

    提示

    如果使用pulsar-daemon start standalone命令將該服務作為後台進程運行，請使用pulsar-daemon stop standalone命令停止該服務。

    有關更多說明，請參見 pulsar-daemon 。
