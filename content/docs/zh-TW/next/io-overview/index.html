Pulsar 連接器概述

把一些外部系統（例如，數據庫或其它信息系統）結合信息系統一起使用時，信息系統的能夠發揮的功能十分強大。

使用 Pulsar IO connectors，能夠方便地創建、部署和管理與外部系統交互的連接器，例如 Apache Cassandra、Aerospike 等等。

概念

Pulsar IO 連接器有兩種類型：source 和 sink。

下圖說明了 source、Pulasr 和 sink 之間的關係：

Source

Source 將外部系統的數據傳輸到 Pulsar。

常見的 source 包括其它信息系統和流水式數據管道 API。

想要了解 Pulsar 內置 source 連接器的完整列表，參閱 source connector。

Sink

Sink 將 Pulsar 的數據傳輸到外部系統。

常見的 sinks 包括其他信息傳遞系統、SQL 和 NoSQL 數據庫。

了解 Pulsar 內建 sink 連接器的完整列表，參閱 sink connector。

信息處理的保證

Processing guarantees 用於處理向 Pulsar topic 寫入信息時發生的錯誤。

Pulsar 連接器和 Pulsar Functions 使用的 processing guarantees 的機制是一致的 。

傳遞語義說明
at-most-once發送到連接器的信息被處理一次或者不被處理。
at-least-once發送到連接器的信息被處理一次或者處理多於一次。
effectively-once發送到連接器的信息有效地被處理一次。

source 和 sink 連接器實現的 Processing guarantees 不僅依賴於 Pulsar guarantee 機制，還與外部系統相關聯。

  • Source：Pulsar 確保遵從 processing guarantees 機制向 Pulsar topic 寫入信息。 Source 連接器受 Pulsar 控制。

  • Sink：processing guarantees 機制依賴於 sink 端的實現。如果 sink 連接器的實現不以冪等方式處理重試，則這個 sink 連接器則沒遵守 processing guarantees 。

設定

創建一個連接器實例時，可以使用以下信息處理保證語義設置:

  • ATLEAST_ONCE

  • ATMOST_ONCE

  • EFFECTIVELY_ONCE

如果在創建連接器時，沒有指定 --processing-guarantees，則為語義為 ATLEAST_ONCE。

以 Admin CLI 為例。了解更多關於 REST API 或 JAVA Admin API 的信息，參閱 here 。

Source
Sink
$ bin/pulsar-admin sources create \
  --processing-guarantees ATMOST_ONCE \
  # Other source configs

了解更多關於 pulsar-admin sources create 的訊息，參閱 here。

$ bin/pulsar-admin sinks create \
  --processing-guarantees EFFECTIVELY_ONCE \
  # Other sink configs

了解更多關於 pulsar-admin sinks create 的訊息，參閱 here。

更新

創建連接器後，可以使用以下語義更新 processing guarantee：

  • ATLEAST_ONCE

  • ATMOST_ONCE

  • EFFECTIVELY_ONCE

以 Admin CLI 為例。了解更多關於 REST API 或 JAVA Admin API 的信息，參閱 here。

Source
Sink
$ bin/pulsar-admin sources update \
  --processing-guarantees EFFECTIVELY_ONCE \
  # Other source configs

了解更多關於 pulsar-admin sources update 的訊息，參閱 here。

$ bin/pulsar-admin sinks update \
  --processing-guarantees ATMOST_ONCE \
  # Other sink configs

了解更多關於 pulsar-admin sinks update 的訊息，參閱 here。

使用連接器

You can manage Pulsar connectors (for example, create, update, start, stop, restart, reload, delete and perform other operations on connectors) via the Connector Admin CLI with sources and sinks subcommands.

連接器（sources 和 sinks）和 Functions 都是以元件實例運行在 Functions workers 上。 通過 Connector Admin CLI 或 Functions Admin CLI 來將source、sink 或 function 的實例啟動在 worker 節點上。 了解更多訊息，參閱 Functions worker。
