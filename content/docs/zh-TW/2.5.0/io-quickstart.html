How to connect Pulsar to database

This tutorial provides a hands-on look at how you can move data out of Pulsar without writing a single line of code.

通過練習本教程中的步驟，對於深入了解 Pulsar I/O 的概念很有幫助。

在本教程的最後，你可以：

  • 將 Pulsar 連接到 Cassandra

  • 將 Pulsar 連接到 MySQL

提示

  • These instructions assume you are running Pulsar in standalone mode. 但是，所有 本教程中使用的命令都可以在多節點 Pulsar 集群中使用，而無需進行任何更改。

  • All the instructions are assumed to run at the root directory of a Pulsar binary distribution.

安裝 Pulsar built-in connectors。

在將 Pulsar 連接到資料庫之前，你需要安裝 Pulsar 和所需的內建連接器。

For more information about how to install a standalone Pulsar and built-in connectors, see here.

啟動 Pulsar 單機模式

  1. 在本地運行Pulsar.

    bin/pulsar standalone

    Pulsar 服務的所有組件均按順序啟動。

    You can curl those pulsar service endpoints to make sure Pulsar service is up running correctly.

  2. 檢查 Pulsar 二進制協議端口。

    telnet localhost 6650

  3. 檢查 Pulsar Function 集群。

    curl -s http://localhost:8080/admin/v2/worker/cluster

    範例輸出

    [{"workerId":"c-standalone-fw-localhost-6750","workerHostname":"localhost","port":6750}]

  4. 確保 public 租戶和 default 命名空間是存在的。

    curl -s http://localhost:8080/admin/v2/namespaces/public

    範例輸出

    ["public/default","public/functions"]

  5. 所有內建連接器都應列出來並標示為可用。

    curl -s http://localhost:8080/admin/v2/functions/connectors

    範例輸出

    [{"name":"aerospike","description":"Aerospike database sink","sinkClass":"org.apache.pulsar.io.aerospike.AerospikeStringSink"},{"name":"cassandra","description":"Writes data into Cassandra","sinkClass":"org.apache.pulsar.io.cassandra.CassandraStringSink"},{"name":"kafka","description":"Kafka source and sink connector","sourceClass":"org.apache.pulsar.io.kafka.KafkaStringSource","sinkClass":"org.apache.pulsar.io.kafka.KafkaBytesSink"},{"name":"kinesis","description":"Kinesis sink connector","sinkClass":"org.apache.pulsar.io.kinesis.KinesisSink"},{"name":"rabbitmq","description":"RabbitMQ source connector","sourceClass":"org.apache.pulsar.io.rabbitmq.RabbitMQSource"},{"name":"twitter","description":"Ingest data from Twitter firehose","sourceClass":"org.apache.pulsar.io.twitter.TwitterFireHose"}]

    If an error occurs when starting Pulsar service, you may see an exception at the terminal running pulsar/standalone, or you can navigate to the logs directory under the Pulsar directory to view the logs.

將 Pulsar 連接到 Cassandra

本節示範如何將 Pulsar 連接到 Cassandra。

提示

  • Make sure you have Docker installed. If you do not have one, see install Docker.

  • The Cassandra sink connector reads messages from Pulsar topics and writes the messages into Cassandra tables. 有關更多訊息，請參見 Cassandra sink 連接器。

設置 Cassandra 集群

本範例使用 cassandra Docker 鏡像在 Docker 中啟動單節點 Cassandra 集群。

  1. 啟動一個 Cassandra 集群.

    docker run -d --rm --name=cassandra -p 9042:9042 cassandra

    Note

    在進行下一步之前，請確保 Cassandra 集群正在運行。

  2. 確保 Docker 進程正在運行。

    docker ps

  3. 檢查 Cassandra 日誌以確保 Cassandra 進程按預期地在運行。

    docker logs cassandra

  4. 檢查 Cassandra 集群的狀態。

    docker exec cassandra nodetool status

    範例輸出

    Datacenter: datacenter1
=======================
Status=Up/Down
|/ State=Normal/Leaving/Joining/Moving
--  Address     Load       Tokens       Owns (effective)  Host ID                               Rack
UN  172.17.0.2  103.67 KiB  256          100.0%            af0e4b2f-84e0-4f0b-bb14-bd5f9070ff26  rack1

  5. 使用 cqlsh 連接到 Cassandra 集群。

    $ docker exec -ti cassandra cqlsh localhost
Connected to Test Cluster at localhost:9042.
[cqlsh 5.0.1 | Cassandra 3.11.2 | CQL spec 3.4.4 | Native protocol v4]
Use HELP for help.
cqlsh>

  6. 創建一個 keyspace pulsar_test_keyspace。

    cqlsh> CREATE KEYSPACE pulsar_test_keyspace WITH replication = {'class':'SimpleStrategy', 'replication_factor':1};

  7. 創建一個表 pulsar_test_table。

    cqlsh> USE pulsar_test_keyspace;
cqlsh:pulsar_test_keyspace> CREATE TABLE pulsar_test_table (key text PRIMARY KEY, col text);

配置 Cassandra sink

Now that we have a Cassandra cluster running locally.

在此部分中，你需要配置 Cassandra sink 連接器。

要運行 Cassandra sink 連接器，需要準備一個配置文件，其中包括 Pulsar 連接器運行時需要知道的資訊。

例如，Pulsar 連接器如何找到 Cassandra 集群，Pulsar 連接器用於向其寫入 Pulsar 信息的 keyspace 和 table 名稱等等。

可以通過以下方法之一來創建配置文件。

  • JSON

    {
    "roots": "localhost:9042",
    "keyspace": "pulsar_test_keyspace",
    "columnFamily": "pulsar_test_table",
    "keyname": "key",
    "columnName": "col"
}

  • YAML

    configs:
    roots: "localhost:9042"
    keyspace: "pulsar_test_keyspace"
    columnFamily: "pulsar_test_table"
    keyname: "key"
    columnName: "col"

有關更多訊息，請參見 Cassandra sink 連接器。

創建 Cassandra sink

你可以使用 Connector Admin CLI 創建 sink 連接器並對其執行其它操作。

Run the following command to create a Cassandra sink connector with sink type cassandra and the config file examples/cassandra-sink.yml created previously.

bin/pulsar-admin sinks create \
    --tenant public \
    --namespace default \
    --name cassandra-test-sink \
    --sink-type cassandra \
    --sink-config-file examples/cassandra-sink.yml \
    --inputs test_cassandra

Once the command is executed, Pulsar creates the sink connector cassandra-test-sink.

This sink connector runs as a Pulsar Function and writes the messages produced in the topic test_cassandra to the Cassandra table pulsar_test_table.

將 Pulsar 連接到 MySQL

本節示範如何將 Pulsar 連接到 MySQL。

提示

  • Make sure you have Docker installed. If you do not have one, see install Docker.

  • The JDBC sink connector pulls messages from Pulsar topics and persists the messages to MySQL or SQlite. For more information, see JDBC sink connector.

設置一個 MySQL 集群

本範例使用 MySQL 5.7 docker 鏡像在 Docker 中啟動單節點 MySQL 集群。

  1. 從 Docker Hub 拉取 MySQL 5.7 鏡像檔。

    $ docker pull mysql:5.7

  2. Start MySQL.

    $ docker run -d -it --rm \
--name pulsar-mysql \
-p 3306:3306 \
-e MYSQL_ROOT_PASSWORD=jdbc \
-e MYSQL_USER=mysqluser \
-e MYSQL_PASSWORD=mysqlpw \
mysql:5.7

    提示

    FlagDescriptionThis example
    -d以分離模式啟動容器。/
    -it保持打開 STDIN 狀態，並分配一個互動終端。/
    --rm退出容器時自動將其消滅移除。/
    -name為容器分配一個名稱。This example specifies pulsar-mysql for the container.
    -p將容器的端口對應到主機端口。This example publishes the port 3306 of the container to the host.
    -e設定環境變數This example sets the following variables:
    - The password for the root user is jdbc.
    - The name for the normal user is mysqluser.
    - The password for the normal user is mysqlpw.

    提示

    有關 Docker 命令的更多訊息，請參見 Docker CLI。

  3. Check if MySQL has been started successfully.

    $ docker logs -f pulsar-mysql

    MySQL has been started successfully if the following message appears.

配置 JDBC sink

Now that we have a MySQL running locally.

在此部分中，你需要配置 jdbc sink 連接器。

  1. Add a configuration file.

    要運行 JDBC sink 連接器，需要準備一個 YAML 配置文件，其中包括P ulsar 連接器運行時需要知道的資訊。

    例如，Pulsar 連接器如何找到 MySQL 群集，JDBC URL 是什麼以及 Pulsar連接器用於向其寫入信息數據的資料表。

    Create a pulsar-mysql-jdbc-sink.yaml file, copy the following contents to this file, and place the file in the pulsar/connectors folder.

    configs:
  userName: "root"
  password: "jdbc"
  jdbcUrl: "jdbc:mysql://127.0.0.1:3306/pulsar_mysql_jdbc_sink"
  tableName: "pulsar_mysql_jdbc_sink"

  2. Create a schema.

    Create a avro-schema file, copy the following contents to this file, and place the file in the pulsar/connectors folder.

    {
  "type": "AVRO",
  "schema": "{\"type\":\"record\",\"name\":\"Test\",\"fields\":[{\"name\":\"id\",\"type\":[\"null\",\"int\"]},{\"name\":\"name\",\"type\":[\"null\",\"string\"]}]}",
  "properties": {}
}

    提示

    有關 AVRO 的更多訊息，請參見 Apache Avro。

  1. Upload a schema to a topic.

    This example uploads the avro-schema schema to the pulsar-mysql-jdbc-sink-topic topic.

    $ bin/pulsar-admin schemas upload pulsar-mysql-jdbc-sink-topic -f ./connectors/avro-schema

  2. Check if the schema has been uploaded successfully.

    $ bin/pulsar-admin schemas get pulsar-mysql-jdbc-sink-topic

    The schema has been uploaded successfully if the following message appears.

    {"name":"pulsar-mysql-jdbc-sink-topic","schema":"{\"type\":\"record\",\"name\":\"Test\",\"fields\":[{\"name\":\"id\",\"type\":[\"null\",\"int\"]},{\"name\":\"name\",\"type\":[\"null\",\"string\"]}]}","type":"AVRO","properties":{}}

創建一個 JDBC sink

你可以使用 Connector Admin CLI 創建 sink 連接器並對其執行其它操作。

This example creates a sink connector and specifies the desired information.

$ bin/pulsar-admin sinks create \
--archive ./connectors/pulsar-io-jdbc-2.6.0.nar \
--inputs pulsar-mysql-jdbc-sink-topic \
--name pulsar-mysql-jdbc-sink \
--sink-config-file ./connectors/pulsar-mysql-jdbc-sink.yaml \
--parallelism 1

Once the command is executed, Pulsar creates a sink connector pulsar-mysql-jdbc-sink.

This sink connector runs as a Pulsar Function and writes the messages produced in the topic pulsar-mysql-jdbc-sink-topic to the MySQL table pulsar_mysql_jdbc_sink.

提示

FlagDescriptionThis example
--archiveThe path to the archive file for the sink.pulsar-io-jdbc-2.6.0.nar
--inputsThe input topic(s) of the sink.

Multiple topics can be specified as a comma-separated list.
--nameThe name of the sink.pulsar-mysql-jdbc-sink
--sink-config-fileThe path to a YAML config file specifying the configuration of the sink.pulsar-mysql-jdbc-sink.yaml
--parallelismThe parallelism factor of the sink.

For example, the number of sink instances to run.		1

提示

了解更多關於 pulsar-admin sinks create 的訊息，參閱 here。

The sink has been created successfully if the following message appears.

"Created successfully"

移除一個 JDBC sink

可以使用 Connector Admin CLI 刪除連接器並對其執行其它操作。

This example deletes the pulsar-mysql-jdbc-sink sink connector.

$ bin/pulsar-admin sinks delete \
--tenant public \
--namespace default \
--name pulsar-mysql-jdbc-sink

提示

了解更多關於 pulsar-admin sinks delete 的訊息，參閱 here。

The sink connector has been deleted successfully if the following message appears.

"Deleted successfully"

本範例再次檢查 sink 連接器的訊息。

$ bin/pulsar-admin sinks get \
--tenant public \
--namespace default \
--name pulsar-mysql-jdbc-sink

The results shows that the sink connector does not exist.

HTTP 404 Not Found

Reason: Sink pulsar-mysql-jdbc-sink doesn't exist
